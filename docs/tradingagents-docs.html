TradingAgents

A production-grade, multi-agent LLM framework for quantitative trading research. Deploy specialized AI agents that debate to reach consensus on trading decisions.

Features Guide Quick Start Glossary API Reference

⚡ How It Works in 60 Seconds

① You Enter a Ticker

Type any stock (NVDA, AAPL) or crypto (BTC-USD, ETH-USD) symbol into the search bar and click Analyze.

② AI Agents Research

4 specialized AI analysts — Market, Sentiment, News, and Fundamentals — independently research the asset in parallel.

③ They Debate

A Bull Researcher argues FOR the trade. A Bear Researcher argues AGAINST. This surfaces risks a single AI might miss.

④ You Get a Signal

After risk review, you receive a structured signal: BUY/SELL/HOLD with confidence %, stop-loss, take-profit, and position size.

Who Is This For?

🎯 For Traders

Run comprehensive AI analysis on any ticker. Get structured trading signals with confidence scores, stop-losses, and take-profit levels.

Walk-forward validation, Deflated Sharpe Ratio, ensemble methods, and shadow trading for rigorous strategy testing.

User Guide

Running Your First Analysis

Option 1: Command Line

python main.py --ticker BTC-USD --date 2026-04-21

This runs a complete analysis and outputs the debate transcript and final signal to your terminal.

Option 2: Web Dashboard

  1. Start both backend and frontend (see Quick Start)
  2. Navigate to http://localhost:5173
  3. Enter a ticker (e.g., BTC-USD or AAPL)
  4. Click "Analyze" and watch real-time agent progress

Understanding the Output

📊 Debate Panel

See the Bull vs Bear arguments from each analyst agent. This surfaces conflicting viewpoints that a single model might miss.

🎯 Signal Badge

Final consensus: BUY, SELL, COVER, or UNDERWEIGHT with confidence percentage.

🛡️ Risk Check

Position size recommendation, stop-loss price, and take-profit levels.

Signal Types

Backtesting

Replay Mode (Use Historical Decisions)

Re-run past analyses exactly as they were originally computed. Great for verifying historical performance.

GET /api/backtest/{job_id}?mode=replay
Simulation Mode (Test on Past Dates)

Run fresh analysis on historical dates with data as-of that date. Tests strategy on unseen data.

POST /api/backtest
{
  "ticker": "BTC-USD",
  "dates": ["2024-01-01", "2024-02-01"],
  "mode": "simulation"
}
Hybrid Mode (Mixed)

Use historical decisions when available, simulate missing dates. Balances speed and coverage.

Shadow Trading (Paper Trading)

Record decisions without real capital, then score them against actual outcomes:

# Record a decision
POST /api/shadow/record
{
  "ticker": "BTC-USD",
  "signal": "BUY",
  "confidence": 0.75,
  "stop_loss": 60000,
  "take_profit": 75000
}

# Score all decisions for a ticker
GET /api/shadow/score/BTC-USD

Understanding the Final Decision Card

🎯 Signal

BUY = Open long · SELL = Close long
SHORT = Open short · COVER = Close short
HOLD = No action needed

📊 Confidence

80%+ = High conviction
60–80% = Moderate
Below 60% = Low, proceed with caution

🛡️ Risk Parameters

Stop-Loss = Exit price to limit losses
Take-Profit = Target price for gains
Position Size = Suggested % of capital
R:R Ratio = Reward-to-risk (higher = better)

📜 History

What it does: Browse all past analyses. Every analysis is auto-saved for later review.

Navigate to History → click a ticker → click a date → see the full decision breakdown with agent reports and debate transcripts.

📈 Backtesting

What it does: Test how the AI's decisions would have performed on historical data.

🔄 Replay — "Re-watch past decisions" (fastest, no AI cost)

Uses your saved analysis results. Best for quickly verifying historical performance.

🔬 Simulation — "What would the AI say about the past?" (costs tokens)

Runs fresh AI analysis on historical dates. Best for testing new models or prompt changes.

🔀 Hybrid — "Best of both worlds"

Uses saved results where available, simulates missing dates. Best for filling gaps in your history.

🎯 Scorecard (Shadow Trading)

What it does: Forward-test predictions without risking real money — a "practice mode" that tracks accuracy.

How It Works

1. AI makes a prediction with a confidence level
2. Prediction is recorded with timestamp
3. Score it against what actually happened
4. Build a track record over time

Key Metrics

Win Rate — % of correct predictions
Brier Score — calibration (0=perfect, 0.25=poor)
Calibration — over/underconfident?

⚡ Pulse (4H Auto-Scheduler)

What it does: Runs automated AI analysis every 4 hours, synced to UTC candle closes. Enable it on the Scorecard page, then check Pulse for results.

Runs at 00:05, 04:05, 08:05, 12:05, 16:05, 20:05 UTC. The 5-min offset ensures the 4H candle has fully closed. Ensemble agreement shows when multiple model configurations agree (stronger signal).

🔧 Auto-Tune

What it does: Automatically tests different AI model and parameter configurations to find the best-performing setup for your target ticker. Select a ticker, choose parameters to optimize, and run — it tests against historical data and shows the winning config.

Developer Guide

Project Structure

The codebase is organized around the principle of separation of concerns:

  • Agents contain domain-specific analysis logic
  • Dataflows abstract exchange APIs behind unified interfaces
  • Graph orchestrates agent collaboration via LangGraph
  • Backtesting provides rigorous strategy validation
  • Pulse handles automation and ensemble management

Adding a New Analyst Agent

  1. Create the agent file in tradingagents/agents/analysts/:
from tradingagents.agents.base_analyst import BaseAnalyst

class MacroAnalyst(BaseAnalyst):
    '''Analyzes macroeconomic trends and their impact on asset prices.'''
    
    def __init__(self, config):
        super().__init__(config)
        self.domain = "macro"
    
    async def analyze(self, ticker: str, context: dict) -> dict:
        # Fetch macro data
        inflation = await self.get_inflation_data()
        rates = await self.get_fed_rates()
        
        # Generate research report
        report = await self.llm.generate(
            prompt=f"Analyze macro impact on {ticker} given inflation={inflation}, rates={rates}"
        )
        
        return {
            "analyst": "MacroAnalyst",
            "sentiment": self.parse_sentiment(report),
            "confidence": self.calculate_confidence(report),
            "reasoning": report
        }
  1. Register in the graph in tradingagents/graph/trading_graph.py:

# Add to analyst initialization
self.analysts = [
    MarketAnalyst(config),

    NewsAnalyst(config),
    FundamentalsAnalyst(config),
    MacroAnalyst(config),  # NEW
]

Quantitative Reference

Deflated Sharpe Ratio (DSR)

The Deflated Sharpe Ratio accounts for the fact that testing multiple strategies inflates the observed Sharpe ratio (multiple comparisons problem).

DSR = Φ((SR − E[max SR]) / SE(SR))

Where:

  • SR: Observed Sharpe ratio
  • E[max SR]: Expected maximum Sharpe under null (from n_strategy trials)
  • SE(SR): Standard error of the Sharpe ratio
  • Φ: Standard normal CDF

Implementation: tradingagents/backtesting/walk_forward.py::compute_deflated_sharpe()

Sharpe Standard Error (Lo, 2002)

The standard formula assumes Gaussian returns. Lo (2002) provides a correction for skewness and kurtosis:

SE = √[(1 + ½·SR² − γ₁·SR + (γ₂/4)·SR²) / n]

Where:

  • γ₁: Skewness of returns
  • γ₂: Excess kurtosis (kurtosis − 3)
  • n: Number of observations

Why this matters for crypto: Bitcoin returns typically have skewness ≈ −1.5 and kurtosis ≈ 8. For SR=1.2, n=100:

  • Gaussian SE: 0.104
  • Fat-tail SE: 0.177 (+70% inflation)

Implementation: tradingagents/backtesting/stats.py::sharpe_standard_error()

Walk-Forward Validation

Walk-forward testing prevents lookahead bias by:

  1. Dividing history into n folds
  2. Training on fold i, testing on fold i+1
  3. Aggregating out-of-sample results
flowchart LR subgraph Fold1[Fold 1] T1[Train] --> Te1[Test] end subgraph Fold2[Fold 2] T2[Train] --> Te2[Test] end subgraph Fold3[Fold 3] T3[Train] --> Te3[Test] end Fold1 --> Fold2 --> Fold3 Te1 --> Aggregate[Aggregate Metrics] Te2 --> Aggregate Te3 --> Aggregate

Key metrics per fold:

  • Sharpe ratio (gross and net of costs)
  • Sharpe SE (with skew/kurtosis)
  • Deflated Sharpe Ratio (DSR)
  • Win rate, profit factor
  • Maximum drawdown

Implementation: tradingagents/backtesting/walk_forward.py::WalkForwardValidator

Shadow Trading Scoring

Paper trades are scored using the Brier score for calibration:

Brier = (1/N) Σ(pᵢ − oᵢ)²

Where pᵢ is predicted confidence and oᵢ is outcome (1 for correct, 0 for wrong). Lower is better.

Scoring tiers:

  • Excellent: Brier < 0.15
  • Good: Brier 0.15−0.25
  • Poor: Brier > 0.25 (overconfident)

Glossary

Plain-language definitions of technical terms used throughout TradingAgents.

TermWhat It Means
Brier ScoreA measure of prediction accuracy. Ranges from 0 (perfect) to 1 (worst). Below 0.15 is excellent; above 0.25 means the AI is poorly calibrated. Calculated as the average squared difference between predicted confidence and actual outcome.
Calmar RatioAnnual return divided by maximum drawdown. Higher means better risk-adjusted returns relative to worst-case losses. A Calmar above 1.0 is generally good.
ConfidenceHow strongly the AI agents agree on a signal, expressed as 0–100%. Higher confidence means stronger consensus among the agents.
DSR (Deflated Sharpe Ratio)An adjusted Sharpe ratio that accounts for the fact that testing many strategies inflates apparent performance. It corrects for "data snooping" — the tendency to find patterns that are just noise.
EnsembleRunning the same analysis with multiple AI model configurations simultaneously. When different models agree, the signal is considered more reliable.
Funding RateA periodic payment between long and short traders on crypto perpetual contracts. Positive = longs pay shorts; negative = shorts pay longs. Affects profitability in backtests.
Kelly CriterionA mathematical formula for optimal bet sizing based on win rate and average win/loss size. Maximizes long-term growth. Requires ≥30 historical trades before activating (falls back to fixed 15% otherwise).
Omega RatioThe probability-weighted ratio of gains to losses. Unlike Sharpe, it considers the entire return distribution. Above 1.0 means gains outweigh losses.
Position SizeWhat percentage of your total capital to allocate to a single trade. Smaller sizes = less risk per trade.
PulseThe automated 4-hour analysis scheduler. Runs AI analysis on a fixed schedule without manual intervention.
R:R RatioReward-to-Risk ratio. The potential profit divided by the potential loss. A 2:1 R:R means you could gain $2 for every $1 risked.
RegimeThe current market state — trending up (bullish), trending down (bearish), ranging (sideways), or volatile (high uncertainty). Different regimes may warrant different strategies.
Shadow TradingPaper trading where AI decisions are recorded but no real money is used. Lets you evaluate AI accuracy before committing capital.
Sharpe RatioReturn per unit of risk (volatility). Above 1.0 is good; above 2.0 is excellent. Measures how much return you get for the risk taken.
Sortino RatioLike Sharpe, but only penalizes downside volatility. Better than Sharpe for strategies that have occasional large gains but controlled losses.
SSE (Server-Sent Events)A technology that lets the server push updates to your browser in real-time. Used to show live progress during analysis.
Stop-LossA price level where you exit a trade to prevent further losses. Set automatically by the AI based on market analysis.
Take-ProfitA price level where you exit a trade to lock in gains. Set automatically by the AI based on market analysis.
Walk-Forward ValidationA testing method that prevents "cheating" by always testing on data the AI hasn't seen. Divides history into training and testing periods, rolls forward, and aggregates results.
