# TradingAgents Design Context

## Users

**Primary persona**: Quantitative researchers and algorithmic traders evaluating multi-agent LLM trading strategies. They are experienced with financial data, comfortable with dense information displays, and expect institutional-grade presentation of analysis results.

**Context of use**: Desktop-first, often alongside terminals and notebooks. Users are comparing agent outputs, reviewing backtest metrics, and making research-informed decisions — not executing trades directly. Sessions range from quick signal checks to deep multi-hour backtest reviews.

**Job to be done**: Run multi-agent AI analysis on equities and crypto assets, review agent reasoning and debates, evaluate backtest performance with statistical rigor, and iterate on strategy configurations.

## Brand Personality

**Three words**: Intelligent, Sleek, Authoritative

**Voice & tone**: Confident but not arrogant. Data speaks first. Copy is concise, precise, and avoids marketing fluff. Technical terms are used correctly without over-explanation. Warnings and caveats (e.g. sample size, backtest limitations) are stated plainly.

**Emotional goals**: The interface should evoke analytical confidence and trust. Users should feel they are working with a serious research tool, not a toy. Every pixel should reinforce credibility.

## Aesthetic Direction

**Visual tone**: Dark, information-dense, modern fintech. Clean typography with strong hierarchy. Glass-morphism cards provide depth without clutter.

**Theme**: Dark mode only (navy-950 `#050816` base).

**References**:
- Stripe Dashboard — clean, modern, beautifully typeset information density
- Linear — sleek dark UI, excellent spacing and hierarchy
- Bloomberg Terminal — data density and professional trust (but with modern aesthetics)

**Anti-references**:
- Cluttered legacy finance UIs (Yahoo Finance, old Bloomberg web)
- Generic SaaS dashboards with oversized cards and too much whitespace
- Flashy DeFi dashboards with excessive neon gradients and animations

- **Library**: Framer Motion
- **Entry animations**: fade-up (`opacity: 0, y: 20` → `opacity: 1, y: 0`)

- **Progress bars**: `duration: 0.5, ease: easeOut`
- **Expand/collapse**: `duration: 0.2`
- **Respect `prefers-reduced-motion`**: Required (WCAG AA)

## Design Principles

1. **Data density over decoration** — Every element must earn its pixels. Prefer compact, information-rich layouts over spacious marketing-style designs. Charts, metrics, and agent outputs should be scannable at a glance.

2. **Hierarchy through typography, not chrome** — Use font weight, size, and color to establish hierarchy. Avoid heavy borders, shadows, or decorative elements. The glass-morphism layer provides subtle depth; don't stack more.

3. **Semantic color is sacred** — Green/cyan = positive/bullish, red = negative/bearish, amber = caution/neutral, purple = short/contrarian. Never use these colors decoratively outside their semantic meaning.

4. **Progressive disclosure** — Show summaries first, details on demand. Agent reports collapse by default. Backtest metrics show headline numbers with drill-down. Respect the user's attention budget.

5. **Credibility through precision** — Display exact numbers, not approximations. Show confidence intervals, sample sizes, and statistical caveats. Never round away meaningful data. The UI should feel like a research instrument.

## Accessibility (WCAG AA)

- **Contrast**: Minimum 4.5:1 for normal text, 3:1 for large text against navy backgrounds
- **Keyboard**: All interactive elements must be keyboard-navigable with visible focus indicators
- **Screen readers**: Semantic HTML, ARIA labels on icon-only buttons, live regions for streaming updates
- **Reduced motion**: Respect `prefers-reduced-motion: reduce` — disable Framer Motion animations, use instant transitions
- **Color independence**: Never convey information through color alone — always pair with text labels or icons (signal badges already do this correctly)

## Component Patterns

- **Cards**: `.glass` class — `bg-glass`, `backdrop-blur-12px`, `border-glass-border`, `rounded-xl`
- **Buttons (primary)**: `bg-gradient-to-r from-accent-teal to-accent-cyan text-navy-950 font-bold`
- **Buttons (secondary)**: `bg-navy-800 border-white/5 text-slate-300 hover:border-accent-teal/30`
- **Inputs**: `bg-navy-800 border-white/10 focus:border-accent-teal/50 focus:ring-accent-teal/20`
- **Badges**: Rounded-full, semantic color with 20% opacity bg + 30% border + full text color
- **Icons**: Lucide React, 4-5px sizes, color matches adjacent text
